# Urban Disaster Monitor Dataset (YOLOv11)
Source: https://universe.roboflow.com/ufrnprojects-xlut9/urban-disaster-monitor
License: CC BY 4.0

## File Information

This dataset is organized in standard YOLO format, with one image file and one label file per sample.

### Root files

- `data.yaml`: Dataset configuration file used by YOLO training scripts.
  - Defines dataset paths (`train`, `val`, `test`)
  - Defines class names and class count (`nc`)
  - Stores Roboflow export metadata
- `README.dataset.txt`: Short dataset guide (this file)
- `README.roboflow.txt`: Original export notes generated by Roboflow

### bSplit folders

- `train/`: Training split (2,674 images + 2,674 labels)
- `valid/`: Validation split (383 images + 383 labels)
- `test/`: Test split (183 images + 183 labels)

Each split contains:

- `images/`: Input images (`.jpg`)
- `labels/`: YOLO annotation files (`.txt`)

### Label file format (`labels/*.txt`)

Each row in a label file represents one object:

`class_id x_center y_center width height`

- Values are normalized to the range `[0, 1]`
- Coordinates follow YOLO convention relative to image width/height
- If an image has multiple objects, its label file contains multiple rows

### Class mapping (from `data.yaml`)

- `0`: cat
- `1`: civilian
- `2`: cow
- `3`: dog
- `4`: horse
- `5`: rescuer

### How to match files

- Image and label files share the same base filename.
  - Example: `train/images/example.jpg` -> `train/labels/example.txt`
