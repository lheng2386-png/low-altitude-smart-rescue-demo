# 第二阶段：风险评分、救援优先级排序与初版报告生成

本步骤在 AeroRescue-AI（面向低空应急救援的无人机多模态灾情识别与辅助决策系统）现有 Gradio + YOLOv11 灾害目标检测 Demo 基础上，增加轻量级决策层能力。不接入外部平台化架构、模型对比模块、语义分割模块，也不调用大模型 API。

## 新增模块

- `app/risk_engine.py`：根据目标类别、检测置信度和目标面积计算初版风险分数。
- `app/priority_ranker.py`：对检测目标按风险分数从高到低排序。
- `app/report_generator.py`：基于检测结果和排序结果，用模板生成中文救援报告。

## 风险评分公式

```text
risk_score = class_weight * 70 + confidence * 20 + area_weight * 10
```

其中：

- `confidence` 为 YOLO 检测置信度，限制在 0 到 1。
- `area_weight = bbox_area / image_area`，限制在 0 到 1。
- 风险分数限制在 0 到 100。

风险等级：

- `0-40`：Low
- `40-70`：Medium
- `70-100`：High

## 类别权重设计理由

```text
civilian: 1.0
horse: 0.65
cow: 0.65
dog: 0.55
cat: 0.55
rescuer: 0.15
```

- `civilian` 代表潜在受困人员，救援优先级最高。
- `horse`、`cow` 属于大型动物，转移难度和现场处置成本较高。
- `dog`、`cat` 属于小型动物，仍纳入救援关注目标。
- `rescuer` 通常代表现场救援力量，默认风险较低，主要用于态势识别。

## 当前局限

当前版本只基于目标类别、检测置信度和目标面积做启发式风险评估，尚未理解目标周围环境风险。例如水域深度、道路阻断、建筑损毁、火情、烟雾、障碍物和可达性等因素暂未纳入评分。

## 下一步计划

第三阶段建议接入 RescueNet 语义分割能力，补充水域、道路阻断、建筑损毁、车辆、可通行区域等环境风险因素，并将目标检测结果与灾区语义分割结果进行空间融合，形成更可靠的综合救援优先级。
