# Contributing to Urban Disaster Monitor

Thank you for your interest in contributing! 🚀

## How to Contribute

### 1. Fork the repository

Create your own fork and clone it locally.

### 2. Create a branch

Use a clear naming convention:

- feat/... → new features
- fix/... → bug fixes
- docs/... → documentation
- exp/... → experiments

### 3. Make your changes

- Follow the project structure
- Keep code clean and documented
- Ensure reproducibility when working with models

### 4. Commit messages

Use clear and descriptive messages:

```
fix: correct dataset split issue
docs: update model evaluation results
```

### 5. Open a Pull Request

Include:

- Description of changes
- Metrics (if model-related)
- Dataset version used
- Comparison with previous versions

---

## Guidelines

- Do not upload large files directly (models/datasets)
- Keep experiments organized
- Ensure reproducibility
- Respect dataset licensing (CC BY 4.0)

---

## Code of Conduct

Be respectful and collaborative. This project aims to support research and real-world impact.
